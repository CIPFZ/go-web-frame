## 当前定位

`plugin` 模块是一套完整的插件项目管理、版本发布审核、工单池认领处理和公开展示能力，不是单一 CRUD 页面。

当前阶段的正式设计重点：

1. 明确业务边界和一期范围。
2. 明确插件项目、发布单、工单池、兼容产品、事件流五类核心对象。
3. 明确“审核员主动认领工单”的流程约束。
4. 为后续后端模块落地、前端页面接入和测试设计提供统一依据。
